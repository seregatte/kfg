# Architecture

Este documento descreve a arquitetura interna do KFG, como os componentes se relacionam e os princípios de design.

## Visão Geral

KFG é um compilador shell declarativo escrito em Go que transforma manifests YAML em funções bash. A arquitetura segue um padrão de pipeline em várias etapas:

```
┌─────────────┐
│   YAML      │
│  Manifests  │
└──────┬──────┘
       │
       ▼
┌─────────────────┐
│  Parser &       │
│  Validator      │
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│  Dependency     │
│  Resolver       │
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│  Kustomize      │
│  Processor      │
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│  Shell Code     │
│  Generator      │
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│  Bash Runtime   │
│  (source/exec)  │
└─────────────────┘
```

## Componentes Principais

### 1. Parser & Validator

**Localização**: `src/internal/parser/`

**Responsabilidades**:
- Ler manifests YAML
- Validar schema (apiVersion, kind, metadata, spec)
- Converter para estruturas internas

**Tipos de Recursos**:
- `Cmd`: Função shell
- `CmdWorkflow`: Orquestração de cmds
- `Step`: Unidade de trabalho reutilizável
- `Schema`: JSON Schema Draft-07
- `Assets`: Dados tipados
- `Converter`: Transformação de assets

**Exemplo de validação**:
```go
type Cmd struct {
    APIVersion string   `yaml:"apiVersion" validate:"required,eq=kfg.dev/v1alpha1"`
    Kind       string   `yaml:"kind" validate:"required,eq=Cmd"`
    Metadata   Metadata `yaml:"metadata" validate:"required"`
    Spec       CmdSpec  `yaml:"spec" validate:"required"`
}
```

### 2. Dependency Resolver

**Localização**: `src/internal/resolver/`

**Responsabilidades**:
- Construir DAG (Directed Acyclic Graph) de dependências
- Ordenar steps por peso (weight)
- Detectar ciclos
- Resolver condicionais (when)

**Algoritmo**:
```
1. Coletar todos os steps do workflow
2. Construir grafo de dependências
3. Ordenar topologicamente (Kahn's algorithm)
4. Aplicar ordenação por weight (estável)
5. Validar condicionais
```

**Exemplo de DAG**:
```
checkout (weight: -100)
    ↓
test (weight: -90, when: checkout.STATUS == "ok")
    ↓
build (weight: -80, when: test.STATUS == "passed")
    ↓
deploy (cmd, weight: -70)
```

### 3. Kustomize Processor

**Localização**: `src/internal/kustomize/`

**Responsabilidades**:
- Processar kustomization.yaml
- Aplicar overlays
- Mesclar recursos
- Resolver referências

**Fluxo**:
```
base/
├── kustomization.yaml
└── resources/
    ↓
overlay/
├── kustomization.yaml (references base)
└── patches/
    ↓
Merged YAML
```

**Integração com Kustomize**:
KFG usa a biblioteca `sigs.k8s.io/kustomize/api` para processar kustomizations. Isso garante compatibilidade total com o ecossistema Kubernetes.

### 4. Shell Code Generator

**Localização**: `src/internal/shellgen/`

**Responsabilidades**:
- Gerar código bash a partir de manifests
- Implementar sistema de cache
- Gerenciar variáveis de ambiente
- Gerar funções de logging

**Estrutura do código gerado**:
```bash
#!/usr/bin/env bash
# Generated by KFG v0.0.8
# DO NOT EDIT

# Global state
export KFG_BUILD_RESULT_FILE="/tmp/kfg-build-XXXXXX.yaml"

# Helper functions
_kfg.log.info() { ... }
_kfg.cache.store() { ... }
_kfg.cache.restore() { ... }

# Step functions
_kfg.step.checkout() {
    # Check cache
    if _kfg.cache.restore "checkout_v1"; then
        return 0
    fi
    
    # Execute
    git clone "$REPO_URL" /tmp/build
    local STATUS="checked_out"
    
    # Store in cache
    _kfg.cache.store "checkout_v1" "$STATUS"
    
    # Export output
    export KFG_STEP_checkout_STATUS="$STATUS"
}

# Cmd functions
hello() {
    _kfg.step.checkout
    echo "Hello, World!"
}
```

**Placeholders**:
```
Template: {env:VAR_NAME}
    ↓ (Go stage)
Generated: $VAR_NAME
    ↓ (Shell runtime)
Runtime: valor_da_variável
```

### 5. Bash Runtime

**Localização**: N/A (código gerado)

**Responsabilidades**:
- Executar funções shell geradas
- Gerenciar estado global
- Implementar cache em runtime
- Logging estruturado

**Variáveis Globais**:
```bash
KFG_BUILD_RESULT_FILE  # Arquivo com resultado do build
KFG_STORE_DIR          # Diretório do store (~/.kfg/store)
KFG_VERBOSE            # Nível de verbosidade (0-5)
KFG_STEP_<name>_<out>  # Output de steps
```

## Package-Oriented Architecture

KFG segue uma arquitetura orientada a pacotes:

```
packages/
├── framework/              # Primitivas compartilhadas
│   ├── manifests/          # Steps reutilizáveis
│   │   ├── materialize/    # Materialização de assets
│   │   ├── cleanup/        # Limpeza de recursos
│   │   └── detect-agent/   # Detecção de AI agents
│   └── tests/              # Testes do framework
│
└── domains/
    └── ai-agents/          # Domínio de AI agents
        ├── manifests/      # Recursos específicos
        │   ├── commands/   # Comandos de agents
        │   ├── steps/      # Steps de agents
        │   └── workflows/  # Workflows de agents
        ├── overlays/       # Overlays por ambiente
        │   └── dev/        # Overlay de desenvolvimento
        └── tests/          # Testes do domínio
```

**Princípios**:
- **Framework**: Primitivas genéricas e reutilizáveis
- **Domains**: Pacotes específicos por domínio de negócio
- **Overlays**: Composição de base + customizações
- **Tests**: Testes em cada nível (framework, domain, integration)

## Data Flow

### Fluxo de Aplicação

```
1. kfg apply -k overlays/dev --workflow agents
   ↓
2. Parser lê kustomization.yaml
   ↓
3. Kustomize processa overlays
   ↓
4. Resolver monta DAG de dependências
   ↓
5. ShellGen gera código bash
   ↓
6. Código é sourceado no shell atual
   ↓
7. Funções ficam disponíveis
```

### Fluxo de Execução

```
1. User executa: agents
   ↓
2. Workflow function é chamada
   ↓
3. Steps executam em ordem (weight)
   ↓
4. Cache é verificado para cada step
   ↓
5. Outputs são exportados como variáveis
   ↓
6. Condicionais (when) são avaliadas
   ↓
7. Cmds executam
   ↓
8. After steps executam (cleanup)
```

## Cache System

**Localização**: `src/internal/cache/`

**Arquitetura**:
```
~/.kfg/store/cache/
├── <hash1>/
│   ├── metadata.json
│   └── data
├── <hash2>/
│   ├── metadata.json
│   └── data
└── ...
```

**Hash Calculation**:
```
hash = SHA256(step.run + step.env + step.output)
```

**Invalidação**:
- Manual: `KFG_REFRESH=1`
- Automática: Mudança no manifest (hash diferente)
- Temporal: `kfg sys cache prune` (> 30 dias)

## Logging System

**Localização**: `src/internal/logging/`

**Arquitetura**:
```
Logs → JSONL file (~/.local/state/kfg/logs/kfg.jsonl)
     ↓
     Console (baseado em KFG_VERBOSE)
```

**Níveis**:
```
0: Quiet (sem output)
1: Error + Warn + Info
2: + Detail
3: + Warn/Detail
4: + Debug
5: + Debug verbose
```

**Estrutura JSONL**:
```json
{
  "timestamp": "2026-06-08T10:30:00Z",
  "level": "info",
  "component": "shellgen",
  "session_id": "abc123",
  "message": "Generated 5 functions"
}
```

## Extension System

**Localização**: `packages/framework/manifests/extensions/`

**Arquitetura**:
```
base/extensions/<name>/
├── kustomization.yaml
├── assets/
│   └── mcp.yaml         # Configuração MCP
└── steps/
    └── install.yaml     # Step de instalação
```

**Fluxo**:
```
1. Extension declarada em kustomization
   ↓
2. Assets carregados (MCP config)
   ↓
3. Step de instalação executado
   ↓
4. Extension disponível para uso
```

## Design Principles

### 1. Declarative Over Imperative

**Ruim** (imperativo):
```bash
#!/bin/bash
git clone repo
cd repo
make test
if [ $? -eq 0 ]; then
    make build
    if [ $? -eq 0 ]; then
        make deploy
    fi
fi
```

**Bom** (declarativo):
```yaml
kind: CmdWorkflow
spec:
  before:
    - step: checkout
    - step: test
      when: checkout.STATUS == "ok"
    - step: build
      when: test.STATUS == "passed"
  cmds: [deploy]
```

### 2. Composition Over Inheritance

```yaml
# base/cmd.yaml
kind: Cmd
metadata:
  name: base.cmd.deploy
spec:
  run: kubectl apply -f manifests/

---
# overlay/patch.yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
patches:
  - target:
      kind: Cmd
      name: base.cmd.deploy
    patch: |
      - op: add
        path: /spec/env
        value:
          ENV: production
```

### 3. Cache by Default

Steps são cacheadas automaticamente quando marcadas como cacheable:

```yaml
kind: Step
metadata:
  annotations:
    kfg.dev/cacheable: "true"
```

### 4. Fail Fast

Validação acontece no parsing, não em runtime:

```go
// Parser valida schema
if manifest.APIVersion != "kfg.dev/v1alpha1" {
    return fmt.Errorf("unsupported apiVersion: %s", manifest.APIVersion)
}
```

## Performance Considerations

### Parsing

- **O(n)** onde n = número de manifests
- Paralelo: cada arquivo parseado em goroutine separada

### Dependency Resolution

- **O(V + E)** onde V = steps, E = dependências
- Algoritmo de Kahn para ordenação topológica

### Shell Generation

- **O(n)** onde n = número de funções geradas
- Template engine otimizado (text/template)

### Cache

- **O(1)** para lookup (hash-based)
- **O(n)** para prune (iterar entradas)

## Security Considerations

### 1. Input Validation

Todos os manifests são validados antes de processamento:
- Schema validation
- Type checking
- Required fields

### 2. No Code Injection

Placeholders são resolvidos de forma segura:
```go
// Ruim: injeção de código
code := fmt.Sprintf("echo %s", userInput)

// Bom: escape adequado
code := fmt.Sprintf("echo %q", userInput)
```

### 3. Sandboxing

Steps executam em subshells isolados:
```bash
_kfg.step.my_step() (
    # Subshell - variáveis não vazam
    local VAR="value"
    echo "$VAR"
)
```

## Testing Strategy

### Unit Tests

**Localização**: `src/internal/*_test.go`

```go
func TestParser_ValidManifest(t *testing.T) {
    manifest := `
apiVersion: kfg.dev/v1alpha1
kind: Cmd
metadata:
  name: test.cmd.hello
spec:
  run: echo "hello"
`
    cmd, err := parser.ParseCmd([]byte(manifest))
    assert.NoError(t, err)
    assert.Equal(t, "test.cmd.hello", cmd.Metadata.Name)
}
```

### Integration Tests

**Localização**: `tests/bats/`

```bash
@test "apply generates shell code" {
    run kfg apply -f test.yaml --workflow test
    [ "$status" -eq 0 ]
    [[ "$output" =~ "test_function()" ]]
}
```

### E2E Tests

**Localização**: `packages/*/tests/`

Testes completos com manifests reais e workflows complexos.
